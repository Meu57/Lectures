Pillar 1: Buniyaadi Parichay aur Storage Layer

Is section ka uddeshya data storage ke buniyaadi formats aur structures ko samajhna hai. Yahan aap Text vs Binary files aur unke organization methods (Sequential vs Direct) ko interactively compare kar sakte hain, jo conventional storage ki foundation hain.

1. Text Files vs. Binary Files

  • Representation: Data ko characters (alphabets, numbers, symbols) ke roop mein store kiya jata hai.
  • Encoding: ASCII, UTF-8 ka upyog. Character 'A' ko ASCII code 65 ke roop mein store kiya jata hai.
  • Readability: Human-readable. Notepad ya VS Code mein asani se padha ja sakta hai.
  • Size & Efficiency: Size bada hota hai. Processing slow hoti hai kyunki numerical calculations ke liye parsing lagti hai.
  • EOF: Special character (jaise Control-Z) se EOF represent hota hai.

Conventional File System Focus
  • • High redundancy (Multiple physical copies).
  • • No dynamic lock controls (Prone to lost updates).
  • • High-level custom query code required.
  • • Limited to general OS file level permissions (R-W-X).
Centralized DBMS Architecture
  • • Systematically controlled using Normalization rules.
  • • Robust Two-Phase Locking and Timestamping protocols.
  • • Native optimized Declarative SQL Engines.
  • • Fine-grained column, cell-level security (Views schema).

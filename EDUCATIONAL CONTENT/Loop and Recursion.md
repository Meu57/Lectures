    # Constant operation c (jaise attendance sheet par ek tick lagana)

Har ek single step par computer jo mehnat kar raha hai use hum ek constant work $c$ bolte hain (jaise $1$ microsecond ka CPU cycle).Hum is pure execution work ko mathematical terms mein kaise likhenge?$$\text{Total Time } T(n) = c \text{ (1st step)} + c \text{ (2nd step)} + c \text{ (3rd step)} + \dots + c \text{ (nth step)}$$Ab isi plus-plus-plus ko hum Sigma $\sum$ ke socket mein fit kur dete hain:$$T(n) = \sum_{i=0}^{n-1} c$$Iska plain English mein matlab kya hua? Maan lo hamare paas ek nested loop hai (loop ke andar loop).for i from 0 to n-1:
    for j from 0 to n-1:
        # Constant operation c

                      [Total = 4 deliveries]

                  ───> Deliver Copy 2 (j = 1)
                  ───> Deliver Copy 3 (j = 2)
                  ───> Deliver Copy 4 (j = 3)

(The Chain of Waiting)Ab aate hain us mudde par jahan standard teaching system fail ho jata hai. Books  bade-bade words likhti  hein: "Recursion is a function that calls itself."Student ke dimaag mein confusion ka dher lag jata hai: "Bhai, function khud ko kaise call kar sakta hai? Hamara computer dimaag naturally jaanta hai ki $5!$ nikalne ke liye humein pure numbers ko multiply karna hai. Lekin is badi problem ko solve karne ke liye computer ise chhote steps mein todta hai:To find factorial(5) ───> We need to calculate factorial(4) first.
To find factorial(4) ───> We need to calculate factorial(3) first.
To find factorial(3) ───> We need to calculate factorial(2) first.
To find factorial(2) ───> We need to calculate factorial(1) first.

Dhyan se dekho! Har ek step par problem ka size chhota hota ja raha hai:$$5 \to 4 \to 3 \to 2 \to 1$$Aur jag hum factorial(1) par pahunchenge, toh humein direct pata hai ki $1! = 1$ hota hai. Is point par hum aage badhna STOP kar dete hain!Computer Science ki bhasha mein:Self-calling Chain: Ek bade size ki problem apne se ek chhotey size ki same problem par depend karti hai. (Recursion Call).Size Reduction: Har step par input size ek se kam ho raha hai ($n \to n-1$).The Stop Condition (Base Case): Woh sabse aakhiri point jahan problem itni chhoti ho jati hai ki humein direct answer pata hota hai (jaise $n = 1$). Computer Science mein isi stop condition ko hum Base Case bolte hain!Agar hum is simple process ka code likhein, toh yeh kitna natural dikhega:def factorial(n):
    # Base Case: Agar n == 1 hai, toh direct answer return karo aur ruk jao!
    if n == 1:
        return 1
    
    # Recursive Call: Agar n > 1 hai, toh ek chhotey size (n-1) ka factorial nikaalo
    return n * factorial(n - 1)

Maan lo agar hum is code mein if n == 1 waala base case lagana bhul jayein, toh kya hoga? Computer bina ruke $1 \to 0 \to -1 \to -2$ jaata rahega aur kabhi nahi rukega!        │ Plate 4: [factorial(1)]  │ <── Active (Hits Base Case!)
       ├──────────────────────────┤
       │ Plate 3: [factorial(2)]  │ <── Paused (Waiting for factorial(1) to finish)
       ├──────────────────────────┤
       │ Plate 2: [factorial(3)]  │ <── Paused (Waiting for factorial(2) to finish)
       ├──────────────────────────┤

       └──────────────────────────┘
             CALL STACK (Memory Stretched!)

Aap dekh rahe ho? Jaise-jaise recursion chain andar ki taraf ja raha hai, hamari RAM/Memory ke upar plates (frames) badhti ja rahi hain! RAM elastic ki tarah khinchti chali ja rahi hai. Agar $n$ calls honge, toh memory mein $n$ active plates ban jayengi.Yahi wajah hai ki linear recursion ki Space Complexity $O(n)$ hoti hai, jagki loop ki space complexity hamesha $O(1)$ hoti hai kyunki loop mein bas ek single pointer slide karta hai, koi naye stack frames nahi bante!Phase 2: Hitting the Base Case & Returning (Going Up)Jaise hi factorial(1) trigger hota hai, use base case se direct value $1$ mil jaati hai. Step 2: [factorial(2)] computes 2*1 ─> Returns to [factorial(3)] ───> Plate 2 deleted!
Step 3: [factorial(3)] computes 3*2 ─> Returns to [factorial(4)] ───> Plate 3 deleted!
Step 4: [factorial(4)] finishes ───> Memory completely empty! Elastic shrinked!

Recursion memory ke andar ek elastic ki tarah pehle stretch hota hai aur fir base case touch karke wapas shrink ho jata hai. Is pure process ko computer science mein Recurrence Relation bolte hain.Sabse Pehle Ek Bahut Badi Misunderstanding Door Karte Hain!Bahut se students jag pehli baar ye equation dekhte hain:$$T(n) = T(n-1) + c$$toh unhe lagta hai ki equation galat hai.
